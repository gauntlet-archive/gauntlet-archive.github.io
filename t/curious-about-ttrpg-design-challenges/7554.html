  1. Breaking roleplaying game conventions. I think too few things that are questioned in tabletop roleplaying game design.

1a) Solving mechanics without randomness (dice, cards). I‚Äôve already made progress with this, but I still going back to using randomization as an instinctive first solution. My goal is to create a strategic TTRGP without any random factors.

1b) Writing adventures and roleplaying games in a new way. I think TTRPGs are written poorly, not being able to convey how to play - only what to play with. Why have character creation first when we don‚Äôt know how to play a TTRPG? Separating the three main elements - mechanics, settings and game master chapter - is not the way to go, and we can even see this structure in indie/story games. I haven‚Äôt been able to find any other resources to be inspired from, but I‚Äôm currently writing TTRPGs as boardgame manuals. 1c) Asking questions about what certain basic design elements results in. Should we have skills/moves?   1. Making sure that the framework connects itself from the beginning of play, to the end.
